# Installations

import { OptionTable } from "components/table";

API مدیریت نصب ابزارک به توسعه‌دهندگان این امکان را می‌دهد که کنترل بر روی نصب ابزارک خود در میان کاربران داشته باشند. این مستندات شامل جزئیات مربوط به هر یک از عملیات موجود، پارامترهای مورد نیاز و نمونه‌هایی از درخواست‌ها و پاسخ‌ها است.


---

### نصب ابزارک برای کاربران
برای نصب ابزارک برای یک یا چند کاربر، می‌توانید از درخواست `POST` به آدرس زیر استفاده کنید:

```http
POST /api/v1/apps/:app_key/installations
```

`:app_key` کلید منحصر به فرد ابزارک شما.

  options={[
    ["user_ids", "String[]", "لیستی از شناسه کاربران"]

/>

مثال درخواست:

```json
{
  "user_ids": ["user1", "user2", "user3"]
}
```

---

### لغو نصب ابزارک برای کاربران
برای لغو نصب ابزارک از روی حساب یک یا چند کاربر، از درخواست `DELETE` به آدرس زیر استفاده کنید:

```http
DELETE /api/v1/apps/:app_key/installations
```

`:app_key` کلید منحصر به فرد ابزارک شما.

#### پارامترهای ورودی

<OptionTable
  options={[
    ["user_ids", "String[]", "لیستی از شناسه کاربران"]
  ]}
/>

مثال درخواست:

```json
{
  "user_ids": ["user1", "user2"]
}
```

---

###  دریافت لیست کاربران نصب‌شده
برای دریافت لیست تمامی کاربرانی که ابزارک شما را نصب کرده‌اند، از درخواست GET به آدرس زیر استفاده کنید:

```http
GET /api/v1/apps/:app_key/installations
```

`:app_key` کلید منحصر به فرد ابزارک شما.

#### پارامترهای خروجی

  ]}
/>

مثال خروجی:

```json

```


### احراز هویت
تمامی درخواست‌ها باید شامل هدر `X-API-KEY` با یک توکن معتبر باشند.
